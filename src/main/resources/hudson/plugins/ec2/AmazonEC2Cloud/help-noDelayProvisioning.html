By default Jenkins do estimate load to avoid over-provisioning of cloud nodes. With this option enabled, a new node is created on Amazon EC2 as soon as NodeProvisioner detects need for more agents. In worse scenarios, this will results in some extra nodes provisioned on EC2, which will be shortly terminated. EC2 being billed by the second since 2017 this won't have a huge impact on infrastructure costs, while offering build nodes without unnecessary delays.
